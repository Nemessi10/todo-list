## 1\. Компонент Автентифікації (`/auth`)

Цей компонент відповідає за реєстрацію, вхід користувачів у систему, оновлення токенів, скидання паролів та вихід з системи.

  * **`POST /auth/register`**: Реєстрація нового користувача.

      * **Призначення**: Створення нового облікового запису користувача.
      * **Запит**:
          * **Метод**: `POST`
          * **URL**: `/api/v1/auth/register`
          * **Тіло (JSON)**:
            ```json
            {
              "name": "Ім'я Користувача",
              "email": "newuser@example.com",
              "password": "newsecurepassword"
            }
            ```
          * **Очікувана відповідь (201 Created)**: Інформація про створеного користувача (без пароля) або статус успіху. Може включати токени авторизації.
          * **Можливі помилки (400 Bad Request, 409 Conflict)**: Некоректні дані, користувач з таким ім'ям/email вже існує.

  * **`POST /auth/login`**: Вхід користувача.

      * **Призначення**: Автентифікація існуючого користувача та отримання токенів доступу.
      * **Запит**:
          * **Метод**: `POST`
          * **URL**: `/api/v1/auth/login`
          * **Тіло (JSON)**:
            ```json
            {
              "email": "user@example.com",
              "password": "securepassword123"
            }
            ```
          * **Очікувана відповідь (200 OK)**: Токен доступу (JWT), можливо, токен оновлення та інформація про користувача.
          * **Можливі помилки (400 Bad Request, 401 Unauthorized)**: Некоректні дані, неправильні облікові дані.

  * **`POST /auth/refresh-token`**: Оновлення токена доступу.

      * **Призначення**: Отримання нового токена доступу за допомогою дійсного токена оновлення.
      * **Запит**:
          * **Метод**: `POST`
          * **URL**: `/api/v1/auth/refresh-token`
          * **Тіло (JSON)**:
            ```json
            {
              "refreshToken": "your_jwt_refresh_token"
            }
            ```
          * **Очікувана відповідь (200 OK)**: Новий токен доступу.
          * **Можливі помилки (401 Unauthorized)**: Недійсний або прострочений токен оновлення.

  * **`POST /auth/forgot-password`**: Запит на скидання пароля.

      * **Призначення**: Ініціювання процесу скидання пароля шляхом надсилання інструкцій на email користувача.
      * **Запит**:
          * **Метод**: `POST`
          * **URL**: `/api/v1/auth/forgot-password`
          * **Тіло (JSON)**:
            ```json
            {
              "email": "user@example.com"
            }
            ```
          * **Очікувана відповідь (200 OK або 202 Accepted)**: Повідомлення про успішне надсилання інструкцій (може не містити тіла).
          * **Можливі помилки (400 Bad Request)**: Невалідний email.

  * **`POST /auth/reset-password`**: Встановлення нового пароля.

      * **Призначення**: Встановлення нового пароля користувача після успішного проходження процесу скидання.
      * **Запит**:
          * **Метод**: `POST`
          * **URL**: `/api/v1/auth/reset-password`
          * **Тіло (JSON)**:
            ```json
            {
              "resetToken": "some_reset_token",
              "newPassword": "newstrongpassword"
            }
            ```
          * **Очікувана відповідь (200 OK або 204 No Content)**: Пароль успішно змінено.
          * **Можливі помилки (400 Bad Request, 401 Unauthorized)**: Недійсний токен скидання, некоректний новий пароль.

  * **`POST /auth/logout` (опціонально)**: Вихід користувача.

      * **Призначення**: Інвалідація токена на сервері (залежить від реалізації).
      * **Запит**:
          * **Метод**: `POST`
          * **URL**: `/api/v1/auth/logout`
          * **Заголовки**: `Authorization: Bearer <токен>`
          * **Очікувана відповідь (204 No Content)**: Успішний вихід.
          * **Можливі помилки (401 Unauthorized)**: Недійсний токен.

## 2\. Компонент Нотаток (`/notes`)

Цей компонент відповідає за створення, отримання, оновлення та видалення нотаток користувача. Всі запити до цього компонента вимагають автентифікації.

  * **`GET /notes`**: Отримання списку нотаток.

      * **Призначення**: Отримання всіх нотаток поточного користувача з можливістю пагінації та сортування.
      * **Запит**:
          * **Метод**: `GET`
          * **URL**: `/api/v1/notes`
          * **Заголовки**: `Authorization: Bearer <токен>`
          * **Параметри запиту (query parameters)**: `limit`, `offset`, `sort`.
          * **Очікувана відповідь (200 OK)**: Масив об'єктів нотаток у форматі JSON.
          * **Можливі помилки (401 Unauthorized, 500 Internal Server Error)**: Неавторизований доступ, помилка на сервері.

  * **`POST /notes`**: Створення нової нотатки.

      * **Призначення**: Створення нової нотатки для поточного користувача.
      * **Запит**:
          * **Метод**: `POST`
          * **URL**: `/api/v1/notes`
          * **Заголовки**: `Authorization: Bearer <токен>`, `Content-Type: application/json`
          * **Тіло (JSON)**: `{ "title": "Заголовок нотатки", "content": "Вміст нотатки" }`
          * **Очікувана відповідь (201 Created)**: Об'єкт створеної нотатки у форматі JSON. Заголовок `Location` з URL новоствореного ресурсу.
          * **Можливі помилки (400 Bad Request, 401 Unauthorized, 500 Internal Server Error)**: Некоректні дані, неавторизований доступ, помилка на сервері.

  * **`GET /notes/{id}`**: Отримання конкретної нотатки.

      * **Призначення**: Отримання нотатки за її унікальним ідентифікатором.
      * **Запит**:
          * **Метод**: `GET`
          * **URL**: `/api/v1/notes/{id}` (де `{id}` - UUID нотатки)
          * **Заголовки**: `Authorization: Bearer <токен>`
          * **Очікувана відповідь (200 OK)**: Об'єкт нотатки у форматі JSON.
          * **Можливі помилки (401 Unauthorized, 404 Not Found, 500 Internal Server Error)**: Неавторизований доступ, нотатку не знайдено, помилка на сервері.

  * **`PUT /notes/{id}`**: Оновлення існуючої нотатки.

      * **Призначення**: Оновлення інформації про нотатку за її унікальним ідентифікатором.
      * **Запит**:
          * **Метод**: `PUT`
          * **URL**: `/api/v1/notes/{id}` (де `{id}` - UUID нотатки)
          * **Заголовки**: `Authorization: Bearer <токен>`, `Content-Type: application/json`
          * **Тіло (JSON)**: `{ "title": "Новий заголовок", "content": "Новий вміст" }` (можливі часткові оновлення)
          * **Очікувана відповідь (200 OK)**: Оновлений об'єкт нотатки у форматі JSON.
          * **Можливі помилки (400 Bad Request, 401 Unauthorized, 404 Not Found, 500 Internal Server Error)**: Некоректні дані, неавторизований доступ, нотатку не знайдено, помилка на сервері.

  * **`DELETE /notes/{id}`**: Видалення нотатки.

      * **Призначення**: Видалення нотатки за її унікальним ідентифікатором.
      * **Запит**:
          * **Метод**: `DELETE`
          * **URL**: `/api/v1/notes/{id}` (де `{id}` - UUID нотатки)
          * **Заголовки**: `Authorization: Bearer <токен>`
          * **Очікувана відповідь (204 No Content)**: Нотатку успішно видалено (відсутнє тіло відповіді).
          * **Можливі помилки (401 Unauthorized, 404 Not Found, 500 Internal Server Error)**: Неавторизований доступ, нотатку не знайдено, помилка на сервері.

## 3\. Компонент Профілю Користувача (`/users/me` або `/profile`)

Цей компонент дозволяє автентифікованому користувачеві переглядати та змінювати свої особисті дані та налаштування. Всі запити вимагають автентифікації.

  * **`GET /users/me`**: Отримання інформації профілю.

      * **Призначення**: Отримання інформації про поточного аутентифікованого користувача.
      * **Запит**:
          * **Метод**: `GET`
          * **URL**: `/api/v1/users/me` (або `/api/v1/profile`)
          * **Заголовки**: `Authorization: Bearer <токен>`
          * **Очікувана відповідь (200 OK)**: Об'єкт з даними користувача (наприклад, ім'я, email, налаштування).
          * **Можливі помилки (401 Unauthorized, 500 Internal Server Error)**: Неавторизований доступ, помилка на сервері.

  * **`PUT /users/me`**: Оновлення інформації профілю.

      * **Призначення**: Оновлення даних поточного аутентифікованого користувача.
      * **Запит**:
          * **Метод**: `PUT`
          * **URL**: `/api/v1/users/me` (або `/api/v1/profile`)
          * **Заголовки**: `Authorization: Bearer <токен>`, `Content-Type: application/json`
          * **Тіло (JSON)**: `{ "name": "Нове Ім'я", "settings": { "theme": "dark" } }` (можливі часткові оновлення).
          * **Очікувана відповідь (200 OK)**: Оновлений об'єкт користувача.
          * **Можливі помилки (400 Bad Request, 401 Unauthorized, 409 Conflict (наприклад, при зміні email), 500 Internal Server Error)**: Некоректні дані, неавторизований доступ, конфлікт даних, помилка на сервері.

  * **`PUT /users/me/password`**: Зміна пароля.

      * **Призначення**: Зміна пароля поточного аутентифікованого користувача.
      * **Запит**:
          * **Метод**: `PUT`
          * **URL**: `/api/v1/users/me/password` (або `/api/v1/profile/password`)
          * **Заголовки**: `Authorization: Bearer <токен>`, `Content-Type: application/json`
          * **Тіло (JSON)**: `{ "currentPassword": "старий_пароль", "newPassword": "новий_безпечний_пароль" }`
          * **Очікувана відповідь (204 No Content або 200 OK)**: Пароль успішно змінено.
          * **Можливі помилки (400 Bad Request, 401 Unauthorized, 500 Internal Server Error)**: Некоректні дані, неправильний поточний пароль, помилка на сервері.

  * **`DELETE /users/me`**: Видалення облікового запису.

      * **Призначення**: Видалення облікового запису поточного аутентифікованого користувача.
      * **Запит**:
          * **Метод**: `DELETE`
          * **URL**: `/api/v1/users/me` (або `/api/v1/profile`)
          * **Заголовки**: `Authorization: Bearer <токен>`
          * **Очікувана відповідь (204 No Content)**: Обліковий запис успішно видалено.
          * **Можливі помилки (401 Unauthorized, 500 Internal Server Error)**: Неавторизований доступ, помилка на сервері.
